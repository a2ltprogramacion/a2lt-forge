# ADR-001: Inicialización de Infraestructura de La Forja

**Fecha:** 11 de marzo de 2026  
**Status:** ACEPTADO  
**Contexto:** Materialización de especificación GEMINI.md v2.3.0  
**Autor:** Ingeniera Arquitecta | **Reviewed by:** Argenis  

---

## Problema

GEMINI.md define un ecosistema completo de agentes y skills, pero carece de:
- Estructura de directorios físicos
- Herramientas de validación operacionales
- Ejemplos funcionales que demuestren el estándar
- Mecanismo de versionado del documento base

Esto crea brecha entre especificación e implementación.

---

## Decisión

**Implementar infraestructura mínima viable en paralelo con especificación:**

1. **Directorios canónicos** (§0.1 GEMINI.md)
   - `./.agent/` (Core)
   - `./catalogo/` (Catálogo)
   - `./output/` (Paquetes)
   - `./rag/` (Search)
   - `./quarantine_lab/` (Sandbox)

2. **Manifiestos iniciales** (§8 GEMINI.md)
   - `./.agent/manifest.json` (Core)
   - `./catalogo/manifest.json` (Catálogo)
   - Esquema JSON validable contra §8.2

3. **Core Toolkit de inicio**
   - `yaml-validator` (v1.0.0): audita frontmatter YAML
   - `component-auditor` (v1.0.0): audita componentes completos

4. **Documentación operativa**
   - `README.md` raíz con quick start
   - `CHANGELOG.md` para tracking de versiones
   - `.gitignore` para evitar commits de artefactos temporales
   - Ejemplo: `output/Ejemplo_Paquete_VCard/GEMINI.md`

5. **Configuración RAG**
   - `rag/config.yaml` con especificación de colecciones
   - Directorios `rag/sources/adrs/` y `rag/sources/sessions/` listos

---

## Rationale

✅ **Ejecución independiente:** Especificación (GEMINI.md) permanece congelada en v2.3.0. Implementación progresa en v2.3.x.

✅ **Reducción de fricción:** Next operador que inicie en La Forja tiene estructura clara + ejemplos de cómo usar.

✅ **Validación temprana:** Los 2 componentes del Core permiten validar el esquema antes de generar más.

✅ **Trazabilidad completa:** CHANGELOG y ADRs registran "por qué" se hace cada decisión.

✅ **Inicio rápido:** README with-ejemplos reduce curva de aprendizaje de días a minutos.

---

## Consecuencias

### ✅ Positivas
- Menos ambigüedad al ejecutar dinámicas §7
- Mecanismo de validación previo a despliegue
- Registro histórico de decisiones arquitectónicas
- Nuevos componentes pueden auto-validarse

### ⚠️ Consideraciones
- Componentes iniciales auditan pero no construyen (eso requiere skill-creator-pro)
- RAG está configurado pero no indexado hasta primer despliegue exitoso
- Ejemplos son mínimos — demuestran formato, no complejidad real

### ❌ No se hace
- No se implementa `skill-search` aún (requiere APIs externas)
- No se implementa UI/dashboard (en roadmap futuro)
- No se configura CI/CD automático (próxima fase)

## Alternativas Consideradas

|---|---|

| Crear componentes de ejemplo en Catálogo en lugar de Core | ✗ Catálogo es solo para producción; ejemplos son referencia |
